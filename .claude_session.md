### Phase 2 тАФ Backend core (done)
- `apps/api/app/llm_client.py` тАФ module-level OpenAI client with `call_chat`, `stream_chat`, `call_json`. Used by `agent.py` and the new intent router.
- `apps/api/app/intent_router.py` тАФ single `classify(message)` entry point. Order: write-regex тЖТ read-deterministic тЖТ tool-detection тЖТ protocol тЖТ LLM JSON extraction тЖТ fallback to LLM. Returns `IntentResult(type, payload)`.
- LLM-driven write extraction kicks in for ambiguous phrasing (uses `call_json` тЖТ `qwen2.5:3b`).
- `apps/api/app/tools.py` тАФ `get_current_time`, `calculator` (safe AST), `get_weather` (wttr.in), `web_search` (DDG instant + HTML scrape fallback), `run_system_command` (whitelist), `get_open_issues`, `create_issue`. `REGISTRY` and `execute_tool(name, args)` for dispatch.
- Streaming `/chat` uses SSE (`text/event-stream`); each token emitted as `data: {"type":"token","content":"..."}`, terminal `{"type":"done", ...}`.
- `apps/api/app/db.py` now exposes `get_db()` context manager. `storage.py` fully refactored to use it.
- New `memories` SQLite table, with `/memory` GET/POST and `/memory/{id}` DELETE. `build_assistant_context` injects last 10 memories into every LLM prompt.
- Structured logging via `logging.basicConfig` + per-request middleware (method, path, status, latency).
- Conversation summary compaction at startup (`_compact_old_summaries`): collapses sessions with тЙе5 summary rows into a single condensed row using the LLM.
